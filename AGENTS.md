# How to work on this repo

Repo purpose: Build the best AI accountability coach.
Default branch: main

Definition of done for any change:
- Make the smallest correct change that improves accountability outcomes.
- Add or adjust tests when behavior changes.
- Run (from repo root):
  - npm test (or project-specific test command)
  - npm run lint (or project-specific lint command)
- If tests fail, fix or revert; never leave main broken.

Preferred work style:
- Open a branch per task.
- Summarize changes, commands run, and results.
- If uncertain about scope or product impact, ask before making risky refactors.

Product principles for this repo:
- Optimize for habit formation, streak safety, and compassionate nudges.
- Prefer transparent, explainable recommendations over black-box scoring.
- Always keep user data privacy and consent explicit.

Release hygiene:
- Keep changes small and reviewable.
- Avoid large dependency bumps unless explicitly requested.
